Linux Basics Lab — Interactive, hands-on practice for beginners

How this lab works

Each section introduces a concept and gives you a task. Type the command in the input box and click Check. We match your answer using simple patterns (regular expressions). There can be multiple correct ways; we accept common variants.

Tip Run these commands on a real Linux system later (Ubuntu, Debian, Fedora, etc.) for muscle memory.

1) Navigation: pwd, ls, cd

Task A: Show your current directory path.

Task B: List all files, including hidden ones, in long format.

Task C: Change directory to the parent folder.

Notes
  • pwd prints the working directory.
  • ls -la shows permissions, owners, sizes, and hidden files.
  • cd .. moves up one level; cd - toggles previous dir.

2) Files & Directories: mkdir, touch, cp, mv, rm

Task A: Create a directory named lab in the current folder.

Task B: Create an empty file named notes.txt inside lab.

Task C: Copy notes.txt to notes.bak.

Task D: Move notes.bak to backup.txt.

Task E (careful): Delete backup.txt.

Notes
  • Use rm -i to confirm deletions; rm -r for directories.
  • Glob patterns like *.txt match many files.

3) View & Search: cat, less, head, tail, grep

Task A: Print the first 5 lines of /etc/passwd.

Task B: Show only lines containing bash from /etc/passwd.

Notes
  • less lets you page through large files; press q to quit.
  • grep -i ignores case; grep -r searches recursively.

4) Permissions: chmod, chown

Task A: Give the owner read/write and group/others read on lab/notes.txt.

Task B: Change owner of lab/notes.txt to user anil.

Notes
  • Mode 644 = owner rw-, group r--, others r--.
  • Use chmod u+x file to add execute for owner.

5) Processes & Services: ps, top, kill, systemctl

Task A: Show all processes of the current user in full format.

Task B: Gracefully stop process with PID 1234.

Task C: Check status of ssh service (systemd).

6) Packages & Editing: apt/dnf, nano, vi

Task A (Debian/Ubuntu): Update package lists.

Task B (Fedora/RHEL): Install curl.

Notes
  • Editors: nano file (easy), vi file (powerful). In vi, type i to insert, Esc, :wq to save & quit.

7) Redirection & Pipes: >, >>, |

Task A: Save ls -l output to out.txt (overwrite).

Task B: Count number of lines in /etc/passwd using a pipe.

8) Environment & Networking: echo, export, ip, ping, curl

Task A: Set an environment variable APP_ENV=dev for the current shell.

Task B: Show your IP addresses.

Task C: Fetch headers from https://example.com.

9) Archive & Disk: tar, gzip, du, df

Task A: Create a compressed tarball lab.tar.gz from lab/.

Task B: Show disk usage of the current directory (human readable).

Task C: Show mounted filesystems usage.

Further Practice

Practice these daily on any Linux VM or WSL. Create a checklist and repeat:

  • Navigation & listing (pwd, ls, cd)
  • File operations (touch, cp, mv, rm)
  • Search & view (grep, less, head/tail)
  • Permissions (chmod, chown)
  • Processes & services (ps, top, kill, systemctl)
  • Pipes & redirection (|, >, >> )
  • Networking (ping, ip, curl)
  • Archives & disk (tar, gzip, du, df)
